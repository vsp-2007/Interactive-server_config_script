🔧 Webmin Samba Share Management Guide

This guide explains how to manage Samba shares using Webmin's web interface.

💡 Key Strategy: We use map to guest = bad user globally. This lets you browse \\IP to see shares. Private shares use Guest Access: No to trigger password prompts.

🔒 Private Share (Password Required)

Step 1: Create Share

Share name: private_vault
Directory to share: /srv/samba/private_vault
Automatically create directory: Yes
Create with owner: smbdata
Create with permissions: 2770
Create with group: smbdata

Step 2: Security & Access Control

Writable: Yes
Guest Access: None (Critical!)
Valid users: your_username

Step 3: File Permission Options

New Unix file mode: 0660
New Unix directory mode: 2770
Force Unix user: smbdata
Force Unix group: smbdata

🌍 Public Share (No Password)

Step 1: Create Share

Share name: public_drop
Directory to share: /srv/samba/public_drop
Automatically create directory: Yes
Create with owner: smbdata
Create with permissions: 2777
Create with group: smbdata

Step 2: Security & Access Control

Writable: Yes
Guest Access: Yes
Guest Unix user: smbdata
Valid users: (Leave Empty)

Step 3: File Permission Options

New Unix file mode: 0666
New Unix directory mode: 2777
Force Unix user: smbdata
Force Unix group: smbdata
⚠️ Troubleshooting: If a folder doesn't ask for password when it should, run net use * /delete /y in Windows CMD to clear cached credentials.
